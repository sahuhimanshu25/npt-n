# Week 8 – Profile Linking Across Social Networks

> Complete Study Notes — NPTEL Social Networks Course

This lecture addresses a practical and technically challenging problem: how to determine whether multiple social media handles (e.g., Facebook, Twitter, LinkedIn) belong to the same real-world person.

---

## The Core Problem and Motivations

Users maintain presence on multiple platforms, often with slightly different handles or details. Linking them is useful for several reasons:

- **Marketing and Advertising** — Avoid wasting budget by sending the same ad to the same person across platforms (e.g., 437k likes on Facebook + 153k followers on Twitter + 800k on LinkedIn may have significant overlap).
- **Sentiment Analysis** — Prevent double-counting opinions. If one person posts positively on Facebook and negatively on Twitter, the true sentiment should reflect a single user, not inflated numbers.
- **Law Enforcement and Security** — Track malicious actors who use different handles across networks for spreading misinformation or illegal content.
- **Research and Personal Curiosity** — Understand your own "social footprint" or verify identities.

The challenge is that users frequently change usernames, profile pictures, bios, or privacy settings. Simple string matching is often insufficient.

---

## Approaches to Profile Linking

### 1. Self-Identification

The strongest signal. Users often explicitly link accounts (e.g., putting a Twitter handle in a Facebook bio, or sharing the same album link across platforms). This provides near-certain evidence of the same person.

### 2. Attribute Comparison

- Profile pictures (visual similarity)
- Bios, locations, education, workplace
- Common links or external websites
- Posting patterns and content style

### 3. Temporal / Historical Analysis

- Track username changes over time. Even if current handles differ, past usernames may overlap or follow similar patterns.
- Monitor when profile details (picture, bio) change and correlate across platforms.

### 4. Graph-Based Features

- Compare friend/follower networks.
- Look for overlapping connections or mutual friends.

### 5. Machine Learning Classification

- Extract 20–50+ features (name similarity, length, character patterns, temporal evolution, Jaccard distance for strings, etc.).
- Train classifiers (e.g., SVM) to output a probability score that two profiles belong to the same person.
- Historical data significantly boosts accuracy.

---

## Key Insights from Research

| Finding | Detail |
|---|---|
| Username changes | 5–7% of users in studied datasets changed handles |
| Profile picture changes | Up to 40% of users changed them multiple times in short periods |
| Link farmers / spammers | Often show high in-degree/out-degree ratios near 1, mimicking normal users |
| ML classifier accuracy | ~76% when combining current + historical data |

---

## Practical Takeaways

Profile linking is not always straightforward. If a user deliberately keeps accounts separate (different pictures, no cross-links, varied bios), it becomes harder. However, most users leave enough overlapping signals for successful matching. These techniques raise serious **privacy concerns** around deanonymization.

This topic ties the entire course together: data collection (APIs), storage (databases), analysis (text processing with NLTK, graph tools like Gephi), and real-world applications in privacy, trust, and security.
