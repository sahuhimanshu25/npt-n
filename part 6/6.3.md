Complete Study Notes: Tutorial on Graph Representation and Social Network Analysis (Twitter Data)
This tutorial teaches how to convert Twitter follower data into a graph structure and perform basic social network analysis using tools like twecoll and Gephi.
Graph Fundamentals
A graph is a data structure made up of nodes (entities) and edges (relationships). In social media:

Nodes represent users, pages, or groups.
Edges represent connections such as "follows", "likes", or "friends".

For a Twitter followee network:

Each node is a user you follow (followee).
A directed edge from user A to user B means A follows B.

This structure helps visualize and analyze how users are connected, who is influential, and how information might flow.
Ways to Represent a Graph
Computers need structured formats to understand graphs. The tutorial covers three common methods:

Adjacency Matrix
A square 2D matrix (size = number of nodes). If there is an edge from node i to node j, the cell at row i, column j is 1; otherwise 0.
It is simple to create in programming but becomes very sparse (mostly zeros and memory-wasting) for large social networks with millions of nodes but relatively few connections.
GraphML
An XML-based format. It uses <graph>, <node>, and <edge> tags. Each node has a unique ID, and each edge specifies source and target nodes. This is the format generated by twecoll and imported into Gephi.
Edge List / CSV
A simple list showing source → target relationships. Easy to generate and read but less rich for complex attributes.

Collecting Twitter Follower Network Data
The tool twecoll (a command-line Python script) is used to fetch data:

Init: Authorize using your Twitter app's Consumer Key and Secret, then enter the PIN.
Fetch: Download your followees (friends) and their followees (friends-of-friends).
Edgelist: Generate the connections and output a .gml file (GraphML format).

This creates a directed graph of your personal network.
Basic Social Network Analysis (SNA) Metrics
These metrics help understand the structure and importance of nodes:

Degree:
In-degree: Number of edges coming into a node (e.g., number of followers).
Out-degree: Number of edges going out from a node (e.g., number of accounts you follow).
Total degree = In-degree + Out-degree.

Centrality (measures importance):
In-degree Centrality: Nodes with the highest number of incoming connections (most followed/influential).
Betweenness Centrality: Nodes that lie on many shortest paths between other nodes (important bridges or connectors).
Closeness Centrality: Nodes with the shortest average distance to all other nodes (can reach others quickly).

Modularity: Measures how strongly the graph is divided into communities (groups of densely connected nodes). Higher modularity means clearer community structure.

Visualization with Gephi
Gephi is a popular open-source tool for exploring and visualizing graphs.
Workflow:

Open the .gml file generated by twecoll.
Use layout algorithms (e.g., Fruchterman-Reingold / ForceAtlas) to arrange nodes nicely (nodes repel, connected nodes attract).
Compute statistics (degree distribution, modularity, etc.).
Customize: Color nodes by modularity class (communities), size nodes by degree (influence), show labels.
Preview and export as image (PNG, SVG, PDF).
Use Data Laboratory tab to view/edit raw node and edge tables.

This turns raw Twitter data into an understandable visual map showing influential users, clusters, and overall network structure.
