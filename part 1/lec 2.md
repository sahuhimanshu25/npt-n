## **Lecture 2: Detailed Study Notes**

### **1. The Gateway: Facebook Developer Portal**

- **The App Concept:** You cannot access the API directly as a user; you must create an **"App"** on the developer portal (`developers.facebook.com`). This App acts as the service requester.
    
- **Graph API Explorer:** A critical testing sandbox. It allows you to simulate API calls, select fields, and preview JSON output without writing a single line of Python code.
    

### **2. The "Key" System: Access Tokens**

- **Definition:** An Access Token is an opaque string that identifies a user, app, or page and defines the **scope** of access.
    
- **Short-lived vs. Long-lived Tokens:**
    
    - **Short-lived:** Typically generated by the Explorer; lasts ~2 hours.
        
    - **Long-lived:** Generated by "debugging" the short-lived token via the **Access Token Debugger**; lasts ~60 days.
        
- **Permissions (Scopes):** Granular access controls (e.g., `user_posts`, `user_birthday`). If a permission isn't explicitly granted, the API call will return an error or empty data.
    

### **3. API Operations & JSON Parsing**

- **The "Me" Object:** A special shortcut in the Graph API that refers to the owner of the current access token.
    
- **Structure of Identifiers:** Facebook often uses a composite key for posts: `[User_ID]_[Post_ID]`.
    
- **Formatting:** Since raw API output is a continuous string, the tutorial recommends `jsonviewer.stack.hu` to visualize the nested hierarchy.
    

### **4. Geographical & Verification Data**

- **Search Parameters:** You can query the graph for "Places" using `center` (latitude/longitude) and `distance` (radius in meters).
    
- **Verification:** The `is_verified` field is a boolean used to distinguish official entities/locations from fan pages.
    

### **5. Implementation in Python**

- **Libraries:**
    
    - `requests`: To send the HTTP GET requests to the Facebook server.
        
    - `json`: To convert the returned string into a Python dictionary (`json.loads()`).
