## 🔤 Handling Encoding Issues

When printing Facebook posts, you may encounter errors like **"ASCII codec can't encode character."** This happens because the default terminal encoding (ASCII) cannot handle special characters, emojis, or non-English text (Unicode).

| Problem | Cause | Solution |
|---|---|---|
| `UnicodeEncodeError: 'ascii' codec can't encode character` | Text contains Unicode characters (emojis, accented letters, non-English scripts) that ASCII cannot represent | Use `.encode('utf-8')` on strings before printing |
| Default terminal encoding is ASCII | ASCII only supports 128 characters (English letters, digits, basic symbols) | UTF-8 is a universal encoding supporting almost all characters worldwide |

> ⭐ **Exam Point:** The "ASCII codec can't encode character" error occurs because the text contains **Unicode characters not supported by default ASCII encoding**. The solution is `.encode('utf-8')`.

```
┌──────────────────────────────────────────────────────────┐
│              Encoding Problem & Solution                  │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Raw Facebook Post: "Great course! 👍 Très bien!"        │
│          │                                               │
│          ▼                                               │
│  print(post)   ← Uses ASCII by default                   │
│          │                                               │
│          ▼                                               │
│  ERROR: 'ascii' codec can't encode character '\U0001f44d'│
│         (the 👍 emoji and 'è' in Très are Unicode)       │
│          │                                               │
│          ▼                                               │
│  FIX: print(post.encode('utf-8'))                        │
│          │                                               │
│          ▼                                               │
│  Output: "Great course! 👍 Très bien!"  ← Works!         │
└──────────────────────────────────────────────────────────┘
```

## 🐍 Introduction to NLTK

NLTK (Natural Language Toolkit) is a powerful Python library for natural language processing. It provides tools for tokenization, stemming, tagging, parsing, and more, along with downloadable corpora and model data.

| Action | Command |
|---|---|
| **Install NLTK** | `sudo pip install nltk` |
| **Download all data** | `nltk.download('all')` |
| **Download specific package** | `nltk.download('punkt')` (for tokenization) |
| **Import tokenizer** | `from nltk import word_tokenize` |
| **Import stemmer** | `from nltk.stem import PorterStemmer` |
| **Import stop words** | `from nltk.corpus import stopwords` |

> ⭐ **Exam Point:** NLTK requires downloading corpora because **basic functions need model data** (e.g., `punkt` for tokenization, `stopwords` for stop word lists). These are not included in the base installation.

```
┌──────────────────────────────────────────────────────────┐
│              NLTK Setup Pipeline                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Step 1: Install NLTK                                    │
│          $ sudo pip install nltk                         │
│          │                                               │
│          ▼                                               │
│  Step 2: Download required data                          │
│          >>> import nltk                                 │
│          >>> nltk.download('punkt')     ← tokenization   │
│          >>> nltk.download('stopwords') ← stop words     │
│          >>> nltk.download('averaged_perceptron_tagger')  │
│          │                              ← POS tagging    │

│          >>> from nltk import word_tokenize              │

│          >>> # ['Hello', 'world']                        │
└──────────────────────────────────────────────────────────┘
```

## 🛠️ Key Text Processing Techniques

### 1. 🔡 Tokenization

Tokenization breaks a sentence into individual words (tokens). This is the foundational step in all text processing — you must split text into units before you can count, stem, tag, or analyze anything.

| Aspect | Detail |
|---|---|
| **What it does** | Splits text into a list of individual words (tokens) |
| **NLTK function** | `word_tokenize(text)` |
| **Required download** | `nltk.download('punkt')` |
| **Best practice** | Always apply `.lower()` first for consistency |

```
┌──────────────────────────────────────────────────────────┐
│              Tokenization Example                         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Input:  "Welcome to NPTEL tutorial"                     │
│          │                                               │
│          ▼  .lower()                                     │
│  Lowered: "welcome to nptel tutorial"                    │
│          │                                               │
│          ▼  word_tokenize()                              │
│  Output: ['welcome', 'to', 'nptel', 'tutorial']          │
│                                                          │
│  Code:                                                   │
│    from nltk import word_tokenize                        │
│    sentence = "Welcome to NPTEL tutorial"                │
│    tokens = word_tokenize(sentence.lower())              │
└──────────────────────────────────────────────────────────┘
```

> ⭐ **Exam Point:** `word_tokenize()` breaks text into a **list of individual words**. Always apply `.lower()` first so that "Course" and "course" are treated as the same word.

### 2. 📊 Frequency Analysis

Use `collections.Counter` to count how often each word appears. This reveals the most common terms in posts (e.g., "course", "nptel", "enrollment").

| Aspect | Detail |
|---|---|
| **What it does** | Counts occurrences of each word in a token list |
| **Python tool** | `collections.Counter` |
| **Best practice** | Remove stop words first to focus on meaningful terms |
| **Example output** | `{'course': 45, 'nptel': 32, 'enrollment': 28}` |

```
┌──────────────────────────────────────────────────────────┐
│              Frequency Analysis Pipeline                   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Tokens: ['welcome', 'to', 'nptel', 'course',           │
│           'nptel', 'course', 'enrollment', 'course']     │
│          │                                               │
│          ▼  Counter(tokens)                              │
│  Result: {'course': 3, 'nptel': 2, 'to': 1,             │
│           'welcome': 1, 'enrollment': 1}                 │
│          │                                               │
│          ▼  .most_common(3)                              │
│  Top 3:  [('course', 3), ('nptel', 2), ('welcome', 1)]  │
│                                                          │
│  Problem: "to" is a stop word — not meaningful           │
│  Solution: Remove stop words before counting             │
└──────────────────────────────────────────────────────────┘
```

### 3. 🌿 Stemming

Stemming reduces words to their root form (e.g., "courses" → "cours", "certification" → "certif"). It uses the **PorterStemmer** from NLTK. Stemming helps consolidate variations of the same word, improving frequency accuracy.

| Aspect | Detail |
|---|---|
| **What it does** | Reduces words to their approximate root form |
| **NLTK tool** | `PorterStemmer` from `nltk.stem` |
| **Important caveat** | Stemmed words are NOT always valid dictionary words |
| **Example** | "courses" → "cours", "running" → "run", "certification" → "certif" |

```
┌──────────────────────────────────────────────────────────┐
│              Stemming Examples                             │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  from nltk.stem import PorterStemmer                     │
│  ps = PorterStemmer()                                    │
│                                                          │
│  "courses"       → ps.stem("courses")       → "cours"   │
│  "certification" → ps.stem("certification") → "certif"  │
│  "running"       → ps.stem("running")       → "run"     │
│  "studies"       → ps.stem("studies")       → "studi"   │
│  "better"        → ps.stem("better")       → "better"   │
│                                                          │
│  Note: "cours" and "certif" are NOT real words!          │
│  Stemming is aggressive — it strips suffixes             │
│  without checking dictionary validity                    │
└──────────────────────────────────────────────────────────┘
```

> ⭐ **Exam Point:** Stemming does **NOT always produce valid dictionary words** (e.g., "cours", "certif"). It is an aggressive approximation that strips suffixes to consolidate word variants.

### 4. 🚫 Stop Word Removal

Common words like "the", "is", "and", "to" appear frequently but carry little meaning. NLTK provides a built-in English stop words list. Removing them helps focus analysis on meaningful content.

| Aspect | Detail |
|---|---|
| **What it does** | Filters out common, low-information words |
| **NLTK tool** | `stopwords.words('english')` from `nltk.corpus` |
| **Examples of stop words** | "the", "is", "and", "to", "a", "in", "of", "it", "for" |
| **Why remove them** | They appear frequently but carry little meaningful information for analysis |

```
┌──────────────────────────────────────────────────────────┐
│              Stop Word Removal Example                    │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Before removal:                                         │
│  ['welcome', 'to', 'the', 'nptel', 'course', 'on',      │
│   'social', 'media', 'and', 'e-crime']                   │
│          │                                               │
│          ▼  Remove stop words (to, the, on, and)         │
│                                                          │
│  After removal:                                          │
│  ['welcome', 'nptel', 'course', 'social', 'media',       │
│   'e-crime']                                             │
│                                                          │
│  → Focus is now on content-bearing words only            │
└──────────────────────────────────────────────────────────┘
```

> ⭐ **Exam Point:** Stop words are removed because they **appear frequently but carry little meaningful information** for analysis. Removing them helps focus on **meaningful / content-specific terms**.

### 5. 🔗 N-grams (Bigrams & Trigrams)

N-grams capture context and phrases that single words miss. They are sequences of N consecutive words extracted from text.

| N-gram Type | Size | Example | Use Case |
|---|---|---|---|
| **Unigram** | 1 word | "online", "course" | Basic frequency analysis |
| **Bigram** | 2 consecutive words | "online course" | Capturing meaningful phrases |
| **Trigram** | 3 consecutive words | "online course enrollment" | Capturing longer contextual phrases |

```
┌──────────────────────────────────────────────────────────┐
│              N-gram Extraction Example                     │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Tokens: ['enroll', 'in', 'online', 'course', 'now']     │
│                                                          │
│  Unigrams (n=1):                                         │
│    'enroll', 'in', 'online', 'course', 'now'             │
│                                                          │
│  Bigrams (n=2):  nltk.ngrams(tokens, 2)                  │
│    ('enroll', 'in'), ('in', 'online'),                   │
│    ('online', 'course'), ('course', 'now')               │
│                                                          │
│  Trigrams (n=3):  nltk.ngrams(tokens, 3)                 │
│    ('enroll', 'in', 'online'),                           │
│    ('in', 'online', 'course'),                           │
│    ('online', 'course', 'now')                           │
│                                                          │
│  → "online course" as a bigram is more meaningful        │
│    than "online" and "course" separately                 │
└──────────────────────────────────────────────────────────┘
```

> ⭐ **Exam Point:** If you want to analyze **phrases instead of single words**, use **N-grams**. Bigrams capture meaningful word pairs and context (e.g., "online course") that single words miss.

### 6. 🏷️ Part-of-Speech (POS) Tagging

POS tagging labels each word with its grammatical role (noun, verb, adjective, etc.). It is useful for understanding sentence structure and performing advanced analysis (e.g., identifying subjects or actions in posts).

| Aspect | Detail |
|---|---|
| **What it does** | Labels each token with its grammatical category |
| **NLTK function** | `nltk.pos_tag(tokens)` |
| **Common tags** | NN (noun), VB (verb), JJ (adjective), RB (adverb), DT (determiner) |
| **Use case** | Understanding sentence structure, identifying subjects/actions, entity extraction |

| POS Tag | Meaning | Example |
|---|---|---|
| **NN** | Noun (singular) | "course", "student" |
| **NNS** | Noun (plural) | "courses", "students" |
| **VB** | Verb (base form) | "enroll", "study" |
| **VBG** | Verb (gerund) | "enrolling", "studying" |
| **JJ** | Adjective | "online", "great" |
| **RB** | Adverb | "quickly", "very" |
| **DT** | Determiner | "the", "a" |
| **IN** | Preposition | "in", "on", "for" |

```
┌──────────────────────────────────────────────────────────┐
│              POS Tagging Example                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Input tokens: ['enroll', 'in', 'the', 'online',        │
│                  'course', 'today']                       │
│          │                                               │
│          ▼  nltk.pos_tag(tokens)                         │
│                                                          │
│  Output:                                                 │
│    ('enroll', 'VB')    ← Verb                            │
│    ('in', 'IN')        ← Preposition                     │
│    ('the', 'DT')       ← Determiner                     │
│    ('online', 'JJ')    ← Adjective                       │
│    ('course', 'NN')    ← Noun                            │
│    ('today', 'NN')     ← Noun                            │
│                                                          │
│  → Reveals grammatical structure of the sentence         │
└──────────────────────────────────────────────────────────┘
```

> ⭐ **Exam Point:** POS tagging helps in **understanding the grammatical structure of sentences** — labelling each word as noun, verb, adjective, etc.

## ⚖️ Comparison: Text Processing Techniques

| Technique | Purpose | Tool / Function | Input | Output | When to Use |
|---|---|---|---|---|---|
| **Tokenization** | Split text into words | `word_tokenize()` | Raw text string | List of tokens | Always — first step in any NLP pipeline |
| **Lowercasing** | Normalize case | `.lower()` | Text string | Lowercase string | Always — before tokenization |
| **Stop Word Removal** | Remove low-information words | `stopwords.words('english')` | Token list | Filtered token list | Before frequency analysis |
| **Stemming** | Reduce to root form | `PorterStemmer().stem()` | Individual token | Stemmed token | When consolidating word variants |
| **Frequency Analysis** | Count word occurrences | `Counter()` | Token list | Frequency dict | To find most common terms |
| **N-grams** | Extract word sequences | `nltk.ngrams(tokens, n)` | Token list | List of n-tuples | To capture phrases and context |
| **POS Tagging** | Label grammatical roles | `nltk.pos_tag()` | Token list | List of (word, tag) tuples | For structural/grammatical analysis |

## 🔄 Practical Workflow Summary

```
┌──────────────────────────────────────────────────────────────┐
│        Complete NLTK Text Processing Workflow                 │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Step 1: FETCH DATA                                          │
│          │  Fetch posts via Facebook Graph API                │

│          │                                                   │

│          │  Unicode characters, emojis, non-English text     │
│          │                                                   │
│          ▼                                                   │
│  Step 3: TOKENIZE & NORMALIZE                                │
│          │  Apply .lower() for case normalization             │
│          │  Apply word_tokenize() to split into tokens       │
│          │                                                   │
│          ▼                                                   │
│  Step 4: REMOVE STOP WORDS & PUNCTUATION                     │
│          │  Filter out "the", "is", "and", "to", etc.        │

│          │                                                   │

│  Step 5: APPLY STEMMING                                      │

│          │                                                   │
│          ▼                                                   │
│  Step 6: ANALYZE                                             │
│          │  Calculate frequencies (Counter)                  │
│          │  Extract n-grams (bigrams, trigrams)              │
│          │  Apply POS tagging                                │
│          │                                                   │
│          ▼                                                   │
│  Step 7: EXTRACT INSIGHTS                                    │
│          │  Most common terms, phrases, or patterns          │
│          │  Sentiment clues, content style analysis          │
│          │  Topic identification                             │
│                                                              │
│  Result: Raw social media text → Structured insights         │
└──────────────────────────────────────────────────────────────┘
```

> ⭐ **Key Takeaway:** The pipeline always follows the order: **Fetch → Encode → Tokenize/Normalize → Remove Stop Words → Stem → Analyze → Extract Insights**.

## 📖 Key Terminology

| Term | Definition |
|---|---|
| **NLTK** | Natural Language Toolkit — Python library for natural language processing |
| **Tokenization** | Splitting raw text into individual words (tokens) |
| **Token** | A single word unit extracted from text |
| **Frequency Analysis** | Counting how often each word/token appears in a corpus |
| **Stemming** | Reducing words to their approximate root form by stripping suffixes |
| **PorterStemmer** | NLTK's implementation of the Porter stemming algorithm |
| **Stop Words** | Common, high-frequency words with little semantic meaning (e.g., "the", "is", "and") |
| **N-gram** | A contiguous sequence of N words extracted from text |
| **Unigram** | Single word (N=1) |
| **Bigram** | Pair of consecutive words (N=2) |
| **Trigram** | Sequence of three consecutive words (N=3) |
| **POS Tagging** | Part-of-Speech tagging — labelling each word with its grammatical role |
| **UTF-8** | Universal character encoding supporting almost all characters worldwide |
| **ASCII** | Basic 128-character encoding (English letters, digits, basic symbols only) |
| **Unicode** | Character standard supporting all writing systems; UTF-8 is its most common encoding |
| **`punkt`** | NLTK data package required for tokenization |
| **`collections.Counter`** | Python class for counting hashable objects (used for word frequencies) |
