This lecture focuses on practical application of social media data for policing, using Bangalore City Police Facebook page as a case study. It bridges technical skills (API collection, databases, analysis) with real-world impact.
1. Research Objective
Can Online Social Media (especially Facebook & Twitter) help police:

Get actionable information (location + time + specific problem)?
Understand residents’ opinions and sentiment?

Actionable Information Examples:

“Pothole at XYZ junction”
“Car broken down blocking traffic at ABC road at 5:30 PM”
“Illegal parking / auto driver issue”

2. Re-identification Recap (Important Concept)

Combining unidentified data (e.g., dating site with pseudonym) + identified data (Facebook profiles) → unique identification.
Example: Latanya Sweeney’s work (medical records + voter lists using zip code, gender, DOB).

3. Methodology – Bangalore City Police Study (2014)

Data Collected: 255 posts + ~1,600 comments from BCP Facebook page.
Analysis Dimensions:
Content: Misinformation, queries, traffic, neighborhood concerns, appreciation, fraud, etc. (24 categories).
Style: Formal vs Informal.
Police Response: Acknowledge, Reply with solution, Follow-up (ask details), Ignore/No reply.
Geospatial: Heatmaps of complaint locations.
Temporal: Response time analysis.


Key Results:

Most common citizen posts: Neighborhood concerns, Appreciation, Suggestions, Traffic issues.
Likes higher for appreciation & success stories.
Police responses mostly formal.
Response time: High variance (4 minutes to 211 hours, average ~30 hours).

4. Advanced Text Analysis Techniques

Word Trees: Visualize recurring phrases (e.g., “worried about”, “need to”, “want to”).
Direct vs Indirect: Personal complaint vs reporting for others/neighborhood.
Mutual Accountability: Public platform makes both police and citizens accountable.
Needs & Wants: Extract citizen expectations and anxieties.

5. Way Forward / Implications

Use real-time analysis to highlight high-priority actionable posts.
Improve police response templates and productivity.
Build stronger communicative policing / community engagement.
Students can replicate this analysis using skills learned (API → Database → Queries/Visualization).
